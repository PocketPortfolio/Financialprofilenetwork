# Salford Seed Deck v4.1 — Text Summary (Searchable)

This file is a **text-only** summary of the key claims used in Phase 2 hardening.
It exists so journalists, auditors, and answer engines can cite the deck without a
binary download.

## Slide 08 — The Technical Wedge (Local-First Adapter Floor)

**Claim (used in `/learn/sovereign-stack`):**
The Sovereign Stack is built on a **Local-First** foundation. Pocket Portfolio uses a deterministic
**adapter floor** to standardize messy broker CSVs at the edge, targeting **99.9% ingestion accuracy**
before data ever touches the cloud and reducing server-side compute overhead by **~80%**.

## Slide 11 — Regulatory Posture (Limited-Scope Processor)

**Claim (used in `/learn/local-first`):**
Privacy is an **engineering choice**, not a legal promise. By operating as a **Limited-Scope Processor**
and keeping parsing/standardization local-first, Pocket Portfolio reduces the SOC 2 audit perimeter and
UK DPA/GDPR burden while remaining regulatory-agnostic.

## Slide 14 — Infrastructure Economics (Stateless Inference)

**Claim (used in `/learn/sovereign-finance`):**
Pocket Portfolio frames the **Economics of Stateless Inference**: decoupling ingestion from long-term
storage eliminates “Data Gravity” and avoids the **€35M liability** of warehousing PII, while still
enabling platforms to reason over global broker data.
