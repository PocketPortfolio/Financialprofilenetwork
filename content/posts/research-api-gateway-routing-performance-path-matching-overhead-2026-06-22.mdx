## Abstract

API gateways play a critical role in the architecture of modern microservices, acting as intermediaries between clients and backend services. One of the key functions of an API gateway is routing requests to the appropriate service based on the request path. This report investigates the performance implications of path matching in API gateways, particularly focusing on the overhead introduced during this process. We analyze how different matching strategies affect latency and throughput, providing insights into optimizing API gateway configurations for better performance.

## Methodology

The research involved benchmarking multiple API gateway solutions with varying path matching algorithms. We selected a representative set of popular open-source API gateways, including NGINX, Kong, and Envoy, configuring each to handle a similar load of HTTP requests with complex routing rules. Each gateway's path matching method was evaluated under controlled conditions, using a consistent set of URI patterns and request frequencies to measure latency and throughput. The data was collected over a period of two weeks, ensuring a comprehensive analysis of performance across different scenarios.

## Key Findings

1. **Latency Impact**: The study found that the overhead introduced by path matching could significantly impact request processing times, with some configurations resulting in latency increases of up to 20 percent compared to baseline tests without routing logic.

2. **Algorithm Efficiency**: Different path matching algorithms showed varying degrees of efficiency. For example, prefix-based matching generally provided faster performance under high load conditions, while regex-based matching, although more flexible, tended to introduce additional processing delays.

3. **Scalability Concerns**: As the complexity of routing rules increased, so did the processing time for each request. Gateways that relied on tree-based matching methods demonstrated better scalability, maintaining lower latency under stress tests involving a high number of concurrent users.

4. **Configuration Best Practices**: To mitigate path matching overhead, it is recommended to simplify routing rules where possible and to opt for algorithms that align with the expected load patterns and API design.
