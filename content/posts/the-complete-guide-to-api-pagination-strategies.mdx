## Problem

APIs often need to serve large datasets efficiently, which can lead to performance issues if not handled correctly. Large data responses can overwhelm both the client and server, leading to slow response times and increased resource consumption. Pagination addresses these challenges by breaking down data into smaller, manageable chunks.

## Solution with Code

Implementing pagination in an API involves deciding how to divide the data. Below are common strategies with examples in Node.js using Express and a sample dataset.

### Offset-Based Pagination

This strategy uses an offset and a limit to determine which records to fetch.

```javascript
app.get('/api/items', (req, res) => {
  const limit = parseInt(req.query.limit) || 10;
  const offset = parseInt(req.query.offset) || 0;
  
  const items = data.slice(offset, offset + limit);
  res.json({
    data: items,
    total: data.length,
    limit: limit,
    offset: offset
  });
});
```

### Keyset Pagination

Also known as "cursor-based" pagination, this strategy uses a unique key (often the primary key) for pagination, which is more efficient for large datasets.

```javascript
app.get('/api/items', (req, res) => {
  const limit = parseInt(req.query.limit) || 10;
  const lastId = req.query.lastId || 0;
  
  const items = data.filter(item => item.id > lastId).slice(0, limit);
  res.json({
    data: items,
    nextPageCursor: items.length ? items[items.length - 1].id : null
  });
});
```

### Page-Based Pagination

This strategy uses page numbers and a specified page size.

```javascript
app.get('/api/items', (req, res) => {
  const page = parseInt(req.query.page) || 1;
  const pageSize = parseInt(req.query.pageSize) || 10;

  const start = (page - 1) * pageSize;
  const items = data.slice(start, start + pageSize);
  res.json({
    data: items,
    currentPage: page,
    totalPages: Math.ceil(data.length / pageSize)
  });
});
```

## Key Concepts

1. **Offset-Based Pagination**: Simple to implement but can be inefficient for large datasets due to the need to skip records.

2. **Keyset Pagination**: More efficient for large datasets, avoiding offset skips, but requires a unique key. Ideal for real-time data where entries are frequently added or removed.

3. **Page-Based Pagination**: User-friendly as it follows the natural concept of pages but can suffer from performance issues similar to offset-based pagination.

Each strategy has its trade-offs and is suited to different use cases. When implementing pagination, consider dataset size, the frequency of data changes, and user experience requirements.
