Building a GraphQL API using TypeScript and Node.js provides a robust framework for creating scalable and type-safe web applications. This guide demonstrates how to set up a basic GraphQL API.

## Problem

REST APIs can lead to over-fetching or under-fetching data, as each endpoint returns a fixed data structure. GraphQL solves this by allowing clients to specify exactly what data they need, promoting efficiency and flexibility.

## Solution with Code

### Step 1: Set Up Your Project

Begin by creating a new Node.js project and installing the necessary dependencies.

```bash
mkdir graphql-api
cd graphql-api
npm init -y
npm install express graphql express-graphql @graphql-tools/schema
npm install typescript ts-node @types/node @types/express --save-dev
```

### Step 2: Configure TypeScript

Create a `tsconfig.json` for TypeScript configuration.

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true
  }
}
```

### Step 3: Define Your GraphQL Schema

Create a `schema.ts` file to define your GraphQL schema.

```typescript
import { makeExecutableSchema } from '@graphql-tools/schema';

const typeDefs = `
  type Query {
    hello: String
  }
`;

const resolvers = {
  Query: {
    hello: () => 'Hello world!',
  },
};

export const schema = makeExecutableSchema({ typeDefs, resolvers });
```

### Step 4: Set Up the Express Server

Create an `index.ts` file to set up the Express server.

```typescript
import express from 'express';
import { graphqlHTTP } from 'express-graphql';
import { schema } from './schema';

const app = express();

app.use('/graphql', graphqlHTTP({
  schema,
  graphiql: true, // Enables the GraphiQL tool for testing
}));

const PORT = 4000;
app.listen(PORT, () => {
  console.log(`GraphQL API server running at http://localhost:${PORT}/graphql`);
});
```

### Step 5: Run Your Server

Use the following command to run your server:

```bash
npx ts-node index.ts
```

### Key Concepts

- **GraphQL Schema**: Defines the types and structure of data available through the API.
- **Resolvers**: Functions that resolve queries to return the requested data.
- **Express-GraphQL**: Middleware that connects GraphQL with an Express server.
- **TypeScript**: Provides type safety, helping catch errors during development.

By following these steps, you have built a basic GraphQL API with TypeScript and Node.js. This setup allows you to expand the API by adding more types and resolvers as needed, ensuring a flexible and efficient data-fetching solution.
