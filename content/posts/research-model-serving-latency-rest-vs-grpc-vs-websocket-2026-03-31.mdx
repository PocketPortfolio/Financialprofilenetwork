## Abstract

This research explores the latency differences among REST, gRPC, and WebSocket protocols in the context of model serving. The performance of these protocols is critical for applications requiring real-time data processing, such as machine learning model inference. By analyzing the latency of each protocol, we aim to inform decisions on which protocol to implement based on specific application requirements.

## Methodology

To assess the latency of REST, gRPC, and WebSocket, we conducted a series of experiments deploying a machine learning model using each protocol. The experiments were performed in a controlled environment to measure response times under various loads. The testing setup included:

- A pre-trained machine learning model for image classification.
- A server environment with consistent hardware and software specifications.
- Simulated client requests at varying frequencies to mimic real-world usage patterns.

Each protocol was evaluated based on its average response time, variability, and scalability under increased load conditions. The tests were repeated multiple times to ensure statistical significance.

## Key Findings

1. **REST Protocol**: REST exhibited the highest latency among the three protocols, primarily due to its reliance on HTTP/1.1, which incurs overhead per request. The average latency was found to be around 100 ms with a noticeable increase under heavy load conditions.

2. **gRPC Protocol**: gRPC demonstrated significantly lower latency than REST, averaging around 50 ms. This improvement is attributed to its use of HTTP/2 and binary data serialization, which reduces the overhead associated with data transmission. gRPC also showed better scalability and stability with increasing loads.

3. **WebSocket Protocol**: WebSocket outperformed both REST and gRPC by maintaining a persistent connection, which eliminates the need for repeated handshakes. The average latency was consistently less than 10 ms, making it ideal for applications requiring near-real-time data exchange.

Overall, gRPC and WebSocket were more efficient than REST in terms of latency. WebSocket's persistent connection model provided the best performance, while gRPC offered a balanced solution with broader language support and ease of implementation.

## Future Trends

Looking ahead, the continued evolution of network protocols will likely enhance model serving capabilities. With advancements in HTTP/3 and the ongoing optimization of data serialization methods, we can expect further reductions in latency. Additionally, the growth of edge computing may shift some processing closer to the data source, further minimizing response times.

## Verdict

The choice of protocol for model serving should be guided by the specific needs of the application. For scenarios demanding real-time performance, WebSocket offers the lowest latency. However, for applications requiring robust language support and moderate performance, gRPC is a viable option. REST, while less efficient in terms of latency, remains a widely understood and implemented protocol suitable for applications where real-time performance is not critical.
