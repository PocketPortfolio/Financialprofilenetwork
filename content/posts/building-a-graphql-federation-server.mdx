## Problem

As applications grow, they often evolve into microservices, each exposing its own GraphQL API. While this provides modularity, it also introduces complexity when combining multiple APIs into a single, cohesive API. Clients must query multiple endpoints, which complicates the client-side code and increases network overhead.

## Solution

GraphQL Federation solves this problem by allowing you to create a single, unified API from multiple GraphQL services. Here's a step-by-step guide to building a GraphQL Federation server using Apollo Server.

### Key Concepts

1. **Federation**: A method for composing multiple GraphQL services into a single, unified graph.
2. **Gateway**: Acts as a unified entry point for executing queries across federated services.
3. **Subgraphs**: Individual GraphQL services that together form the federated graph.

### Step-by-Step Implementation

First, ensure that each of your services is a federated service. This requires them to define a schema with `_Entity` and `_Service` types, typically using Apollo's Federation library.

#### Define a Federated Service

```javascript
// user-service.js
const { ApolloServer, gql } = require('apollo-server');
const { buildFederatedSchema } = require('@apollo/federation');

const typeDefs = gql`
  type User @key(fields: "id") {
    id: ID!
    name: String
  }

  extend type Query {
    user(id: ID!): User
  }
`;

const resolvers = {
  User: {
    __resolveReference(user, { fetchUserById }) {
      return fetchUserById(user.id);
    },
  },
  Query: {
    user(_, { id }, { dataSources }) {
      return dataSources.userAPI.getUserById(id);
    },
  },
};

const server = new ApolloServer({
  schema: buildFederatedSchema([{ typeDefs, resolvers }]),
});

server.listen({ port: 4001 }).then(({ url }) => {
  console.log(`User service ready at ${url}`);
});
```

#### Create the Gateway

Now, set up the Apollo Gateway to unify the federated services.

```javascript
// gateway.js
const { ApolloServer } = require('apollo-server');
const { ApolloGateway } = require('@apollo/gateway');

const gateway = new ApolloGateway({
  serviceList: [
    { name: 'user', url: 'http://localhost:4001/graphql' },
    // Add more services as needed
  ],
});

const server = new ApolloServer({
  gateway,
  subscriptions: false,
});

server.listen({ port: 4000 }).then(({ url }) => {
  console.log(`Gateway ready at ${url}`);
});
```

### Conclusion

By composing multiple GraphQL services into a single federated schema using Apollo Federation, you simplify client interaction and reduce network overhead. This approach allows for a scalable and maintainable system architecture, ensuring that as your application grows, your GraphQL API remains performant and easy to use.
