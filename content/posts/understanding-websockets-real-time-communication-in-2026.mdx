## Problem

In today's digital landscape, real-time communication is crucial for interactive web applications. Traditional HTTP is insufficient for scenarios requiring continuous data exchange, like online gaming, live chats, or financial tickers. HTTP's request-response model is inefficient for these tasks due to latency and overhead.

## Solution with Code

WebSockets provide a solution by establishing a persistent connection between client and server, enabling low-latency, bidirectional communication. Let's explore a simple implementation using Node.js and a front-end JavaScript client.

### Setting up a WebSocket Server

First, create a basic WebSocket server using Node.js with the `ws` library:

```bash
npm install ws
```

Create a file `server.js`:

```javascript
const WebSocket = require('ws');

const server = new WebSocket.Server({ port: 8080 });

server.on('connection', (socket) => {
  console.log('Client connected');
  
  socket.on('message', (message) => {
    console.log(`Received: ${message}`);
    socket.send(`Server: ${message}`);
  });

  socket.on('close', () => {
    console.log('Client disconnected');
  });
});

console.log('WebSocket server is running on ws://localhost:8080');
```

### Implementing a WebSocket Client

Now, create a simple HTML file to connect to this WebSocket server:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>WebSocket Client</title>
</head>
<body>
  <script>
    const socket = new WebSocket('ws://localhost:8080');

    socket.addEventListener('open', (event) => {
      console.log('Connected to server');
      socket.send('Hello Server!');
    });

    socket.addEventListener('message', (event) => {
      console.log(`Message from server: ${event.data}`);
    });

    socket.addEventListener('close', (event) => {
      console.log('Disconnected from server');
    });
  </script>
</body>
</html>
```

## Key Concepts

- **Persistent Connection**: Unlike HTTP, WebSockets maintain an open connection, allowing data to be sent anytime.
- **Bidirectional Communication**: Both client and server can send messages independently.
- **Low Latency**: WebSockets reduce overhead by eliminating the need for HTTP headers with each message, improving speed.
- **Scalability**: WebSockets handle multiple simultaneous connections efficiently, making them suitable for high-load systems.

By implementing WebSockets, you can significantly enhance the real-time communication capabilities of your web applications, ensuring a seamless and interactive user experience.
