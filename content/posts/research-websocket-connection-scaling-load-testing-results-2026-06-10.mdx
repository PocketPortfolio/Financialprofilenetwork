## Abstract

WebSocket technology is crucial for real-time web applications, allowing two-way interactive communication between a client and a server. As businesses increasingly rely on these connections, understanding how WebSockets scale and perform under load is essential. This study investigates WebSocket connection scaling and load testing results, providing insights into performance implications and optimal configurations. By simulating real-world scenarios, this research aims to equip developers with the knowledge to enhance application scalability.

## Methodology

The research involved a series of load tests using Apache JMeter, a popular open-source tool for performance testing. We simulated multiple WebSocket connections to evaluate scalability and performance under stress conditions. The test environment consisted of a cloud-based server setup, using both horizontal and vertical scaling methods to measure the effects on connection stability and throughput. Key performance indicators (KPIs) such as response time, connection stability, and data throughput were monitored and analyzed across varying load levels.

To ensure accuracy, each test was repeated multiple times under controlled conditions. We varied the number of concurrent connections, starting from one thousand and scaling up to one hundred thousand, to observe how different server configurations handled the load. Additionally, we incorporated network latency simulations to mimic real-world internet conditions, providing a comprehensive view of WebSocket scalability.

## Key Findings

1. **Scalability and Performance**: WebSockets demonstrated high scalability potential, maintaining stable connections and low latency even as the number of concurrent connections increased. However, performance varied significantly depending on server configurations and network conditions.

2. **Server Configuration Impact**: Horizontal scaling, involving the addition of more servers, proved more effective at maintaining performance levels compared to vertical scaling, which involved upgrading the existing server's resources. This was particularly evident as connection numbers approached the upper limits.

3. **Network Conditions**: Simulated latency had a noticeable effect on performance, emphasizing the need for robust network infrastructure to support high numbers of WebSocket connections. Under increased latency, response times increased, but connection stability remained largely unaffected.

4. **Optimal Configuration Strategies**: The research highlighted the importance of balancing server resources and network capabilities. For optimal performance, a mix of horizontal scaling and strategic load balancing is recommended, along with regular performance testing to adapt to changing conditions.
