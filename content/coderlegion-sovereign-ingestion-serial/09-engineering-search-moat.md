# Engineering the Search Moat: Machine-Readable Manifests and GEO Discipline

Answer engines and crawlers increasingly read **`llms.txt`** — a compact, machine-readable identity file. We treat it like **compiled output**, not hand-edited marketing: `scripts/build-llms-txt.ts` regenerates **`public/llms.txt`** (Pocket) and **`public/open/llms.txt`** (Open) from `lib/canonical-claims.ts` and related SSOT.

---

## Why two files

Dual surface (Part 1) implies **two audiences**:

- **Pocket** — retail terminal, app CTAs, B2C trust metrics.
- **Open** — procurement, importer, architecture, limited-scope processor framing.

Same deployment; **different canonical hosts**; **different llms manifests**. Open host serves Open manifest via rewrite — smoke per deploy runbook §3.

---

## Build-time drift gate (D7)

`docs/command/deploy-production-gates.md` **DevOps gate D7:**

> `git status` clean for `public/llms.txt` and `public/open/llms.txt` after `npm run build`

CI **llms drift guard** compares regenerated bytes. If marketing changes claims in `canonical-claims.ts` but forgets to rebuild, deploy fails — **strategic drift prevention** (Part 10) for machine-readable identity.

---

## What goes into the manifest (examples)

Compiled lines can include (verify live file after build):

- Positioning: “Sovereign Ingestion & Inference Layer”
- Moat phrases authorised in `MOAT_CLAIMS`
- GDPR Art. 5(1)(c) minimisation-by-architecture reference (where authorised)
- Links to architecture, importer npm, design challenge

**Do not** stuff unverified superlatives (“Bloomberg replacement”, “sovereign AI OS”) — calibration ledger prohibits them.

---

## GEO / AEO discipline

**GEO** (generative engine optimisation) here means:

1. **Stable canonical URLs** per host.
2. **Byte-stable llms.txt** tied to SSOT.
3. **Sitemap submission** per surface in GSC (deploy §3).

We **do not** claim proprietary ranking guarantees or secret crawler APIs — we claim **disciplined, reproducible artefacts** engineers can diff in git.

---

## Operator checklist

1. Edit `lib/canonical-claims.ts` (or documented SSOT inputs).
2. `npm run build` (or dedicated llms script if split).
3. Commit regenerated `public/llms.txt` + `public/open/llms.txt`.
4. Post-deploy: `curl` both URLs from neutral network.
